# Часть один

## Постановка задачи

Разработать модель компьютерного зрения для детекции и классификации мелких плавающих объектов на изображениях, полученных с БПЛА над водной поверхностью. Задача необходима для систем мониторинга акваторий в целях безопасности (поиск утопающих), экологического контроля (обнаружение мусора) и навигации.

## Формат входных и выходных данных

Вход: RGB-изображение (цветное), размер переменный (оригинальные аэроснимки), как правило, от 1280×720 до 3840×2160 пикселей. Формат: JPEG/PNG (тензор [H, W, 3], значения 0–255).
Выход: список обнаруженных объектов. Каждый объект описывается:
bbox: [x_center, y_center, width, height] — нормализованные координаты (float от 0 до 1), где:
x_center, y_center — центр ограничивающего прямоугольника
width, height — ширина и высота прямоугольника
label: целочисленный ID класса (например, 0 – person, 1 – boat, 2 – debris...)
confidence: float от 0 до 1

## Метрики

Основные метрики:

- mAP@0.5 (mean Average Precision при IoU=0.5) – стандарт для детекции.
- mAP@[0.5:0.95] (среднее по IoU от 0.5 до 0.95) – более строгий вариант.

Ожидаемые значения (бейзлайн / целевая модель):

- Бейзлайн (YOLOv5n без дообучения на AFO): mAP@0.5 ~0.30–0.40 (много ложных срабатываний на бликах/волнах)
- Целевая модель (YOLOv8 или Faster R-CNN с тонкой настройкой): mAP@0.5 ~0.70–0.85
- Recall по классу "person": ≥0.90 (критично для спасения)

## Датасеты

Источник: `AFO - Aerial dataset of floating objects` на Kaggle.
Ссылка: https://www.kaggle.com/datasets/jangsienicajzkowy/afo-aerial-dataset-of-floating-objects
Авторы: Gąsienica-Józkowy, Jan and Knapik, Mateusz and Cyganek, Boguslaw
Размер: 4.7 GB
Дата публицакации: 2021 год

Объём и структура (по документации датасета):

- Количество изображений: 3647
- Количество аннотаций: 39991 объектов
- Разрешение: от 1280×720 до 3840×2160
- Классы: люди в воде, лодки, спасательные круги, мусор, надувные плоты и пр.
- Разметка: Darknet YOLO format (нормализованные `[x_center, y_center, width, height]`)
- Разделение по объектам: train 67,4% / val 13,48% / test 19,12%

Особенности и проблемы:

- Объекты очень мелкие (от 5×5 до 50×50 пикселей)
- Сильные блики, отражения, волны
- Разная высота и угол съёмки
- Дисбаланс классов (людей мало по сравнению с лодками/мусором)

## Моделирование

### Бейзлайн

Бейзлайн: YOLOv5n (nano) предобученная на COCO, дообученная на AFO всего 10 эпох.  
Предобработка: изменение разрешения на 640x640, приведение в тензор, нормализация  
Постобработка: приведение сырого выходного тензора в словарь с полями boxes, scores, labels, удаление предсказий с уверенностью ниже 1e-3, примение NMS с параметром пересечения 0.5  
Ожидаемый mAP@0.5: ~0.55–0.65.  
Почему бейзлайн: минимальные вычислительные затраты, быстрое развёртывание, возможность измерить прирост от более сложной архитектуры и аугментаций.

### Основная модель

Модель: YOLOv8m (medium)
Способ обучения:

- Инициализация весов с COCO
- Fine-tuning на AFO
- Optimizer: AdamW
- Learning rate: 1e-1 с косинусным затуханием
- Batch size: 32
- Learning rate scheduler: warmup + cosineannealing
- Аугментации: отсутствует
- Размер входного изображения: 640×640
- Число эпох: 20
- Формат модели: сырой checkpoint

### Внедрение

Форма использования:

- Запуск на локальном пк (CPU/GPU) модели в формате torch
- Дополнительные компоненты:
  - NMS (Non-Maximum Suppression) для фильтрации дублирующихся боксов
  - Фильтр по порогу confidence (например, >0.3 для людей, >0.5 для мусора)
  - Лёгкая постобработка для объединения пересекающихся боксов одного объекта
- Целевое устройство: локальный пк (оптимально - с gpu)
